Setting Up Git for a Project



  1. Creating a git project.

    1. Make your project directory.

      • ‘mkdir newproject’

    2. ‘cd’ to your new directory.

      • ‘cd newproject’

    3. Initialise the directory for git.

      • ‘git init’

  2. Create your repository.

    1. Logon to your server.

    2. ‘cd’ to your base git directory.

      • ‘cd /srv/git’

    3. Make a directory for the new repo.

      • ‘sudo mkdir newproject.git’

    4. ‘cd’ to your new directory.

      • ‘cd newproject.git’

    5. Initialise the directory for git.

      • ‘sudo git init --bare’

    6. Edit the text description of the new repo.

      • ‘sudo vim description’

    7. Add the owner to the config file.

      • ‘sudo vim config’

        • Add line ‘[gitweb]’

        • Add line owner = Mark Grant

    8. Change git repo ownership.

      • ‘cd ..’

      • ‘sudo chown -Rv git:www-data newproject.git’

    9. Log off the server.

  3. Link the project directory with the git repo.

    1. Go back to your project directory.

    2. Add the remote repository.

      • ‘git remote add origin git@hermes:/srv/git/newproject.git’

  4. After an initial commit.

    1. Set-up master branch to track the repo.

      • ‘git push --set-upstream origin master’

    2. Create a new working branch.

      • ‘git checkout -b mg’

    3. Set-up mg branch to track the repo.

      • ‘git push --set-upstream origin mg’
