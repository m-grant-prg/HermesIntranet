Build A Package From sid or testing



Before you do this!!

Backports are better than building from testing or sid so look there first.

https://backports.debian.org/Instructions/

https://backports.debian.org/Packages/



Build the package

Using thunar as an example package and testing as an example source:-



  1. Find available packages

    1. Type ‘rmadison thunar --architecture amd64’.

    2. The response will be something like:-

      • thunar | 1.2.3-4+b1 | oldoldstable | amd64

      • thunar | 1.6.3-2 | oldstable | amd64

      • thunar | 1.6.11-1 | stable | amd64

      • thunar | 1.6.15-1 | testing | amd64

      • thunar | 1.8.1-1 | unstable | amd64

  2. Locate the package in testing or sid.

    1. Use package tracker.

      • https://tracker.debian.org/

      • enter thunar.

    2. copy link location of dsc file.

  3. Download the dsc file.

    1. cd to where you want the files stored.

    2. Open a terminal.

    3. Type dget -x and paste the link location.

      • dget -x http://deb.debian.org/debian/pool/main/t/thunar/thunar_1.6.15-1.dsc

  4. Build the package.

    1. Disable the hook for dsc gpg check as we won’t do signing checks.

    2. Perform a usual pbuilder on the dsc file.

    3. Remember to re-establish dsc gpg check hook.

  5. Publish the .deb files to the repo as normal.
