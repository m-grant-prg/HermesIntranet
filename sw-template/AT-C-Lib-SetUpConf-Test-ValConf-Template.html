A C project demonstrating creating a library with bash config file setup, test harness and validate config file source. Also shows support for convenience temporary header files.



Demonstrates:-

  • In configure.ac-

    • The [--enable-debug=<yes|no> [default: no]] option. Enables the appropriate setting of compiler flags.

    • Creating a variable for temporary header file processing.

    • C compiler checks.

    • GNULib macros.

    • Enabling shared libraries.

    • Archiver macro for static libraries.

    • Creating confg.h â€“ header file about the configuration.

    • Program existence checks.

    • Library existence checks.

    • Header file existence checks.

    • Checks for pkg-config files.

  • Create a library.

  • Conditional noinst_HEADERS based on the configure.ac AC_SUBST variable for temporary headers. (e.g. could be temporarily copied from a -dev package).

  • Bash script to setup a config file.

  • Library test harness.

  • C source for validating a config file.

  • Non-AutoTools controlled include file. Could be purely local, or a library header file temporarily moved here from lib-dev package during development.

  • Using a standard version.c source file.
